> **Epic**：OPS-P2-2026Q3  
> **范围**：Amazon 写回 Seller Central · AliExpress 违规纳入顶栏同步 · Temu 竞品自动监控  
> **基线租户**：HangZhouYiTuo（tenant 5）· Temu / AliExpress / Amazon 各 1 店  
> **关联现状**：[amazon-integration/01-需求与范围.md](../amazon-integration/01-需求与范围.md)、[competitor-monitor/01-需求清单.md](../competitor-monitor/01-需求清单.md)

## 1. 背景与问题陈述

| 编号 | 现状 | 用户痛点 | 业务影响 |
|------|------|----------|----------|
| **G1** | Amazon PATCH 写操作仅更新 `amazon_operational_item` 本地 JSON（`AmazonWriteServiceImpl`） | 运营在 CrossHub 点「已回复/已发货」后，Seller Central 仍显示待处理 | 双系统不一致，易漏单、重复劳动 |
| **G2** | AliExpress 顶栏 `platformSync` 只触发 `POST /api/aliexpress/crawl`（订单）；违规需页内单独 `POST /violations/sync` | Boss 顶栏显示「已同步」但违规 Tab 仍空 | 待办漏报，违规 SLA 风险 |
| **G3** | Temu 竞品分析在 Boss Tab 手动 discover + analyze；`/api/monitor` 框架已有但未接入顶栏 | 竞品变化需人工记得进 Tab 点分析 | 监控滞后，爆款/跟卖响应慢 |

## 2. 建设目标

### 2.1 业务目标

1. **Amazon**：CrossHub 写操作在 Seller Central **真实生效**（或明确失败原因），本地状态与平台一致。
2. **AliExpress**：顶栏一键同步后，**订单 + 违规**在同一轮任务中完成（或可配置）。
3. **Temu**：已配置竞品目标在顶栏同步或定时任务中 **自动刷新快照**，Boss 打开 Tab 即见最新分析。

### 2.2 非目标（Out of Scope）

- 替代紫鸟做账号隔离；写操作仍走 **本机 Agent + 紫鸟 WebDriver**（与读同步一致）。
- Amazon SP-API 官方 Messaging/Shipping API 一期不做（可作为 P3 备选，见技术方案）。
- Temu 竞品「全网自动发现新竞店」— 仍须 Boss 首次添加目标 URL/关键词。
- 员工端开放竞品配置（本期仅 Boss 配置 + 全员只读）。

## 4. 功能需求明细

### 4.1 Amazon 写操作（FR-AMZ）

| 编号 | 需求 | 验收标准 |
|------|------|----------|
| FR-AMZ-01 | 保留现有 PATCH API 路径不变 | `PATCH /api/amazon/daily/messages/{id}` 等兼容 |
| FR-AMZ-02 | 写请求改为 **异步 Agent 任务**，本地状态 `pending → running → success/failed` | UI 轮询或 WebSocket；成功前按钮 loading |
| FR-AMZ-03 | Agent 在紫鸟浏览器内完成 DOM 操作并截图留证 | `backend/data/amazon-captures/write_*` |
| FR-AMZ-04 | 平台成功后更新 `amazon_operational_item`；失败保留原状态并写 `error_message` | DB 可追溯 |
| FR-AMZ-05 | 同一店铺写任务与 `amazon_sync` 读任务 **互斥**（409） | 避免浏览器抢占 |
| FR-AMZ-06 | 支持写操作类型：`buyer_message_reply`、`review_handle`、`case_ack`、`outbound_ship` | 见技术方案 payload |

### 4.2 AliExpress 违规顶栏同步（FR-AE）

| 编号 | 需求 | 验收标准 |
|------|------|----------|
| FR-AE-01 | `syncAliExpressStores` 在订单 crawl 成功后 **链式** 触发违规 sync（默认开启） | 顶栏一次点击完成 |
| FR-AE-02 | 或扩展 `POST /api/aliexpress/crawl` body：`scope=all`（orders+violations 单次 Python 进程） | 减少双 job 409 |
| FR-AE-03 | 顶栏 message：`已同步 X 笔订单、Y 条违规` | 与 API 读数一致 |
| FR-AE-04 | 违规 crawl 失败时：订单 success 仍展示 partial，不整站 failed | 对齐 Temu partial 语义 |
| FR-AE-05 | `aliexpress_crawl_job.violations_count` 写入并在 poll 返回 | 已有字段，需顶栏消费 |

### 4.3 Temu 竞品自动监控（FR-TM）

| 编号 | 需求 | 验收标准 |
|------|------|----------|
| FR-TM-01 | 竞品目标入库 `monitor_target`（platform=temu）或沿用 `temu_competitor` + 关联 monitor | 单一数据源 |
| FR-TM-02 | 顶栏 Temu 同步完成后，对 `auto_sync=1` 且 `canCrawlToday` 的目标 POST `/api/monitor/targets/{id}/trigger` | 最多 N 个/次（默认 3） |
| FR-TM-03 | Java Scheduler 支持竞品 `interval_minutes=1440` 默认定时 | 不依赖用户开页 |
| FR-TM-04 | 竞品 Tab 读 monitor latest API，不再仅 localStorage | 后端模式零 Demo |
| FR-TM-05 | profile_busy / 前端登录 required 时 skipped + 指引 | 错误码 `COMPETITOR_FRONTEND_LOGIN_REQUIRED` |

## 5. 约束与依赖

| 项 | 说明 |
|----|------|
| 架构 | Java `:18080` + Python Agent `:18765` + 共用 `crosshub.db`（WAL） |
| Amazon | 紫鸟 `ziniao` 绑定、`integration_agent` 在线 |
| AliExpress | Playwright 无头 crawl，单租户单 job 锁 |
| Temu 竞品 | Temu **前台** profile 与卖家后台 profile 分离；竞品爬前台 |
| 安全 | 写操作仅 Boss/有店铺 scope 的运营；审计 tenant 隔离 |
| 合规 | 写操作频率限制：同店同类型 ≥30s 间隔 |

---

## 6. 成功标准（Epic 级）

- [ ] Amazon：E2E 至少 1 条买家消息回复在 SC 可见（或截图证明 + job success）
- [ ] AliExpress：顶栏同步后 `/api/aliexpress/violations` 的 `syncedAt` 非空（有违规时条数 >0）
- [ ] Temu：顶栏同步后至少 1 个已配置竞品生成当日 snapshot（`temu_competitor_snapshot` 或 monitor snapshot）
- [ ] 三特性均 **不破坏** 现有读同步与 Demo 模式开关
- [ ] 回归 tenant 5 三平台读 API 仍 200

## 7. 里程碑建议

| 阶段 | 内容 | 工期估 |
|------|------|--------|
| **M1** | AliExpress 违规链式顶栏同步 | 2～3 天 |
| **M2** | Temu 竞品 monitor 接入顶栏 + Tab 联调 | 4～5 天 |
| **M3** | Amazon 写回 Agent 任务（消息+发货 P0） | 7～10 天 |
| **M4** | Amazon 评论/Case + 审计 | 3～5 天 |

**推荐实施顺序**：M1 → M2 → M3（M1 改动最小、价值立现；M3 复杂度最高）。
