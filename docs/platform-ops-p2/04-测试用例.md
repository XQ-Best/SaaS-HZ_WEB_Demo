## A. AliExpress 违规顶栏同步（M1）

### A1. 功能 — 顶栏链式同步

| ID | 用例 | 步骤 | 期望 | 优先级 |
|----|------|------|------|--------|
| AE-TC01 | 顶栏同步含违规 | 1. Boss 登录 2. 顶栏点「同步全部」或仅 AliExpress | AE 项 `status=success` 或 `partial`；message 含「违规」字样 | P0 |
| AE-TC02 | violations API 更新 | 顶栏同步完成后 `GET /api/aliexpress/violations` | `syncedAt` 为当日；若有平台违规则 `violations.length>0` | P0 |
| AE-TC03 | 订单仍正常 | 同步后 `GET /api/aliexpress/orders/today` | 订单数与同步前一致或更新 | P0 |
| AE-TC04 | scope=all 单 job | DB 查最新 `aliexpress_crawl_job` | 仅 1 条 running/success；无并发双 job | P0 |
| AE-TC05 | 顶栏 rowCount | 查看顶栏 AE 店铺项 | `rowCount >= 订单数`（含违规计数逻辑） | P1 |

### A2. 异常与 partial

| ID | 用例 | 步骤 | 期望 | 优先级 |
|----|------|------|------|--------|
| AE-TC10 | 违规失败 partial | Mock Python violations 段失败 | 顶栏 `partial`；订单 Tab 仍有数据 | P1 |
| AE-TC11 | crawl 409 | 页内手动 crawl 进行中再点顶栏 | `skipped` 或等待，不 stack 多个 failed | P1 |
| AE-TC12 | 后端离线 | 停 Java 后顶栏同步 | `skipped`，不误报 success | P1 |

### A3. 模块页一致性

| ID | 用例 | 步骤 | 期望 | 优先级 |
|----|------|------|------|--------|
| AE-TC20 | 页内刷新违规 | AliExpress 模块 → 违规 Tab → 刷新 | 与 API 一致；不依赖顶栏 | P2 |
| AE-TC21 | 确认申诉 PATCH | 选一条违规 → 确认申诉 | `PATCH /violations/{id}` 200；列表状态更新 | P2 |

## B. Temu 竞品自动监控（M2）

### B1. Monitor 目标与顶栏

| ID | 用例 | 步骤 | 期望 | 优先级 |
|----|------|------|------|--------|
| TM-TC01 | 添加竞店 | Boss → Temu → 竞品 Tab → 添加 URL/发现 | `monitor_target` 或等价表有记录 | P0 |
| TM-TC02 | 顶栏触发快照 | 设 auto_sync=随顶栏 → 顶栏 Temu 同步 | 当日 snapshot 行存在 | P0 |
| TM-TC03 | 竞品 Tab 展示 | 打开竞品 Tab | 显示产品列表/分析；`last_snapshot_at` 非空 | P0 |
| TM-TC04 | 每日仅一次 | 同日顶栏同步两次 | 第二次不重复 crawl（或 force=false skip） | P1 |
| TM-TC05 | 限流 3 目标 | 配置 >3 个 auto_sync 竞店 | 顶栏最多触发 3 个 job，其余下轮 | P1 |

### B2. 登录与 profile

| ID | 用例 | 步骤 | 期望 | 优先级 |
|----|------|------|------|--------|
| TM-TC10 | 前台未登录 | 未跑 frontend_login 时 trigger | skipped + `COMPETITOR_FRONTEND_LOGIN_REQUIRED` 指引 | P0 |
| TM-TC11 | 卖家 crawl 不阻塞 | 顶栏 Temu 销售 crawl 与竞品并行策略 | 卖家 288 条仍 success；竞品 skipped 或 queued | P1 |
| TM-TC12 | profile_busy | 登录窗口占用时 | 明确 skipped 文案 | P1 |

### B3. 定时调度

| ID | 用例 | 步骤 | 期望 | 优先级 |
|----|------|------|------|--------|
| TM-TC20 | 定时 job | 设 interval=60min，等待 scheduler | `monitor_job` success；snapshot 更新 | P2 |
| TM-TC21 | 手动 force | Tab 点「立即分析」force=true | 忽略 canCrawlToday，强制新 snapshot | P1 |

## C. Amazon 写回 Seller Central（M3/M4）

### C1. 买家消息回复（P0）

| ID | 用例 | 步骤 | 期望 | 优先级 |
|----|------|------|------|--------|
| AMZ-TC01 | 提交回复 | daily 有未回复消息 → 选模板 → 提交 | 返回 `write_job_id`；UI loading | P0 |
| AMZ-TC02 | 写 job 成功 | 轮询 `GET /api/amazon/write/{id}` | status=success；item.payload.status=replied | P0 |
| AMZ-TC03 | SC 真实生效 | Agent 成功后人工查 SC 收件箱 | 该 thread 已回复（或截图证明） | P0 |
| AMZ-TC04 | 写失败 | 关闭紫鸟后提交 | status=failed；error_code 含 ZINIAO/AGENT；本地仍 unreplied | P0 |
| AMZ-TC05 | 写读互斥 | write running 时 POST `/api/amazon/sync` | 409 `AMAZON_SYNC_IN_PROGRESS` 或 write 409 | P0 |

### C2. 出库发货（P0）

| ID | 用例 | 步骤 | 期望 | 优先级 |
|----|------|------|------|--------|
| AMZ-TC10 | 填写运单发货 | outbound Tab → 填 tracking → 发货 | write job success；payload.shipped_at 有值 | P0 |
| AMZ-TC11 | 无效运单 | 空 tracking | 400 校验，不创建 job | P1 |
| AMZ-TC12 | SC 订单状态 | 成功后查 SC unshipped 列表 | 订单不在待发货（或截图） | P0 |

### C3. 评论与 Case（P1）

| ID | 用例 | 步骤 | 期望 | 优先级 |
|----|------|------|------|--------|
| AMZ-TC20 | 处理差评 | reviews Tab → 标记已处理 | write job + payload.handled_at | P1 |
| AMZ-TC21 | 已读 Case | cases Tab → 确认已读 | write job success | P1 |

### C4. 权限与审计

| ID | 用例 | 步骤 | 期望 | 优先级 |
|----|------|------|------|--------|
| AMZ-TC30 | 员工 scope | 员工仅绑定部分店 → 写他店 item | 403/404 | P0 |
| AMZ-TC31 | 审计日志 | Boss 写操作成功后查 audit | 含 user_id、action、job_id、时间 | P2 |
| AMZ-TC32 | 节流 | 30s 内同 item 重复提交 | 429 或忽略第二次 | P2 |
