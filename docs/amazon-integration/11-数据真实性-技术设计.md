# Amazon 运营数据真实性 — 技术设计（V3）

> **版本**：v3.0  
> **日期**：2026-07-13  
> **需求依据**：[10-数据真实性-需求方案.md](./10-数据真实性-需求方案.md)

---

## 1. 设计原则

1. **Provenance-first**：字段入库前带 `source`（页面 key + 解析器版本）
2. **Composer 只 merge，不 invent**：合并多源时禁止生成新数值
3. **Fail-empty**：解析失败 → 留空，不 fallback 到账户级指标
4. **One crawl, one bundle**：`scope=full` 单次紫鸟会话采全平台运营数据
5. **UI 与 DB 一致**：`0` 仅表示后台确实为 0；未知用 `null` / 空字符串 + UI `—`

## 4. 字段契约（CrawlPayload → DB）

### 4.1 `amazon_product_snapshot`

| 列 | 类型 | 空值语义 | 合法 source |
|----|------|----------|-------------|
| `revenue_30d` | TEXT | 空 = 未采集 | `br_child_asin` |
| `orders_30d` | INT | 0 = 后台为 0 或未采集* | `br_child_asin` |
| `page_views` | INT | 0 = 未采集或真 0† | `br_child_asin` |
| `conversion_rate` | REAL | 0 = 未采集或真 0† | `br_child_asin` |
| `ad_spend_30d` | TEXT | 空 = 未采集 | `ads_campaign_manager` |
| `acos` | REAL | 0 = 未采集 | `ads_campaign_manager` |
| `tacos` | REAL | 0 = 未采集 | `derived:tacos`（需 spend+revenue） |
| `inventory` | INT | 0 = 未采集或真 0† | `inventory_all` |

\* 未采集与真 0 的区分在 V3.1 用 `field_sources_json` 细化；V3 先保证不造假。  
† 依赖 `page_diagnostics` 判断是否采集成功。

### 4.2 禁止入库模式（CI 检测）

```python
FORBIDDEN_PATTERNS = [
    lambda rows: all_same_positive(rows, "acos"),          # 全相同 ACOS
    lambda rows: all_spend_equals_revenue_ratio(rows, 0.408),  # 固定比例 spend
]
```

## 5. `scope=full` 页面执行顺序

单次浏览器会话内顺序（减少重复 goto）：

1. `home` → metrics + news/cases 种子  
2. `account_health`  
3. `messages` / `reviews` / `cases`  
4. `br_child_asin`  
5. `inventory_all` + `home_catalog`  
6. `orders_*`（8 子页，去重）  
7. `ads_campaign_manager`  
8. `coupons` / `shipments`  
9. `compose_product_rows` → `merge_campaign_ads` → `enrich_product_rows`（无账户分摊）

超时：整 job 900s（现有 Agent 限制）；单页超时记入 `page_diagnostics`。

## 6. 错误与空态

| 场景 | `error_code` | UI |
|------|--------------|-----|
| 紫鸟未登录 | `AMAZON_LOGIN_REQUIRED` | 指引登录，不展示旧假数据混新数据 |
| BR 0 行 | `AMAZON_NO_PRODUCT_ROWS` | partial + Warning |
| 广告页 0 行 | —（非失败） | SKU 广告列全 `—` |
| Agent 离线 | `AMAZON_AGENT_OFFLINE` | 操作指引 |

**重要**：partial 成功时**不得**用账户 ACOS 补全产品行。

## 9. 风险

| 风险 | 缓解 |
|------|------|
| 真实数据大量 `—`，运营觉得「变空了」 | 产品文案说明「不再显示估算值」；修 parser 提高覆盖率 |
| `full` 单次超时 | 页级 diagnostic + 允许 partial；调优等待时间 |
| Campaign Manager 无 ASIN 列 | 加强 `EXTRACT_AD_SKU_SPEND_JS`；仍无则保持 `—` |
