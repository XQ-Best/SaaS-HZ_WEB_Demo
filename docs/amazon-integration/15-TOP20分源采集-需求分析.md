## 1. 背景与问题陈述

### 1.1 用户反馈（Boss · 产品 TOP20）

| 现象 | 严重度 |
|------|--------|
| 「7日订单」不准确 | P0 |
| 销售额空 | P0 |
| 广告花费、ACOS、TACoS 全空 | P0 |
| 转化率仅个别 SKU 有值 | P1 |
| FBA 库存仅少部分有值 | P1 |
| 「ACOS 偏高」「ACOS 过高」筛选无数据 | P0 |
| 业务规则：ACOS ≥35% 偏高、≥50% 过高（与现 UI 阈值 30/40 不一致） | P2 |

### 1.2 数据库实证（同步后）

最近一次 `scope=reports` 成功同步（2026-07-13 11:01:15）：

| 字段 | 覆盖率（37 SKU） |
|------|------------------|
| `revenue_30d` 非空 | **0 / 37** |
| `ad_spend_30d` 非空 | **0 / 37** |
| `acos > 0` | **0 / 37** |
| `orders_30d > 0` | **4 / 37**（值均为 1，不可信） |
| `page_views > 0` | **2 / 37**（均为 1787，疑似汇总误解析） |
| `conversion_rate > 0` | **2 / 37** |
| `inventory > 0` | 极少 |

`backend/data/amazon-captures/` 存在大量 `br_nodata_a0/a1/a2` 截图 → **Business Report DOM 解析系统性失败**。

### 1.3 根因归类

| 根因 ID | 描述 | 影响字段 |
|---------|------|----------|
| **RC-01** | `crawl_business_report()` 依赖 DOM 虚拟滚表 + JS 抠单元格，BR 页未命中 | 订单、销售额、会话、转化率、TOP 排序 |
| **RC-02** | `reports` scope 使用 `fast=true`，BR 仅 1 次尝试、库存仅 1 页 | 覆盖率下降 |
| **RC-03** | BR 失败后退化为库存/catalog ASIN 列表，`compose_product_rows` 无指标可合并 | 37 行「有名字无数字」 |
| **RC-04** | `crawl_ads_data()` 在 `advertising.amazon.com` DOM 解析，SKU 花费未命中 | 广告花费、ACOS、TACoS、筛选器 |
| **RC-05** | `enrich_product_rows` 在 `ad_spend≤0` 时清空 acos/tacos | ACOS 筛选永远 0 条 |
| **RC-06** | UI 标注「7日」，爬虫选「最近 30 天」，字段名 `orders_30d` | 认知偏差、对账困难 |
| **RC-07** | 无 CSV 导出、无 XHR 抓包、无分源 provenance | 无法定位失败环节 |

### 1.4 与 V3 的关系

V3（数据真实性）已落实 **「宁可空，不可假」**，下架账户级 ACOS 分摊。  
V4 解决 **「空了之后怎么可靠地采到真数」** — 不再强化 DOM 解析，改为 **分源采集（CSV 优先 / 抓包备选）**。

## 2. 建设目标

### 2.1 一句话

**产品 TOP20 的销量、流量、转化来自 Business Report CSV；库存来自库存 CSV 或 SP-API；广告来自 Sponsored Products ASIN 报表 CSV；Playwright 只负责登录与触发导出，不再抠表格 DOM。**

### 2.2 量化目标（YOTO美国账号，BR+广告报表均成功时）

| 指标 | 当前（2026-07-13） | V4 目标 |
|------|-------------------|---------|
| TOP20 行 `revenue` 非空占比 | 0% | **≥ 90%** |
| TOP20 行 `orders` 与 SC 一致 | 不可对账 | **抽 5 ASIN 误差 0** |
| TOP20 行 `sessions` 非空占比 | ~5% | **≥ 85%** |
| TOP20 行 `conversion_rate` 非空占比 | ~5% | **≥ 85%** |
| 有广告投放 SKU 的 `ad_spend` 非空 | 0% | **≥ 80%**（有广告 SKU 子集内） |
| `inventory` 在 TOP20 内非空 | 极低 | **≥ 70%** |
| ACOS 偏高/过高筛选可用 | 否 | **是**（有广告数据时） |
| BR DOM 解析依赖 | 100% | **0%**（TOP20 路径） |
| 同步后 `br_nodata` 截图率 | 高 | **< 5%**（改为 CSV 失败才截图） |

### 2.3 非目标（Out of Scope）

| 项 | 说明 |
|----|------|
| SP-API 全量接入 | P2 可选；V4 以 CSV + 抓包为主 |
| 利润率 / 成本核算 | 仍不展示 |
| 改写 daily 运营待办（消息/差评/Case）采集方式 | 维持现有 DOM，另 Epic |
| 多店铺并行 Agent | 仍串行 per account |
| Walmart / 其他平台 | 不涉及 |

## 4. 功能需求

### 4.1 数据采集（FR-采集）

| ID | 需求 | 验收要点 |
|----|------|----------|
| FR-C01 | Business Report 通过 **CSV 下载** 获取子 ASIN 报表 | 不调用 `EXTRACT_BUSINESS_REPORT_JS` 作为主路径 |
| FR-C02 | BR 日期范围为 **最近 7 天**（与 UI 一致） | `period_days=7` 入库 |
| FR-C03 | 库存通过 **Manage Inventory Export CSV** 或 SP-API 补全 | 按 ASIN merge |
| FR-C04 | 广告通过 **Sponsored Products ASIN 报表 CSV** | 废弃 Campaign Manager DOM 主路径 |
| FR-C05 | BR CSV 失败时，可降级 **XHR 抓包**（单次） | 记录在 `page_diagnostics` |
| FR-C06 | 各源失败独立告警，不阻塞其他源入库 | 部分成功 → job `partial` |
| FR-C07 | 每字段记录 `source`（br_csv / inv_csv / ads_csv / xhr_br） | 见技术方案 |

### 4.2 合成与展示（FR-展示）

| ID | 需求 | 验收要点 |
|----|------|----------|
| FR-D01 | TOP20 按 **7日销售额** 降序 | 与 BR CSV 一致 |
| FR-D02 | `tacos = ad_spend / revenue × 100`（仅当两者均有真实值） | 遵守 V3 R5 |
| FR-D03 | ACOS 阈值：**35% 偏高、50% 过高** | 更新 `amazonBoss.js` 常量 |
| FR-D04 | UI 列名与周期一致：「7日订单」「7日销售额」 | 或改字段为 30 日（二选一，本包选 7 日） |
| FR-D05 | `data_quality` 随 insights API 返回 | 含各源行数、失败原因 |

### 4.3 同步任务（FR-任务）

| ID | 需求 | 验收要点 |
|----|------|----------|
| FR-J01 | `scope=reports` 不再使用 `fast` 跳过关键步骤 | BR/库存/广告均完整尝试 |
| FR-J02 | 新增 `scope=products`（可选）仅跑 TOP20 三源，耗时更短 | 前端「BR 刷新」可指向此 scope |
| FR-J03 | 订单列表 **不** 用于填充 TOP20 销量 | orders-v3 仅给出库 Tab |

## 5. 约束与原则

继承 V3 数据真实性宪法（R1–R7），并追加：

| # | 原则 |
|---|------|
| **V4-1** | TOP20 销量/流量/转化 **只信 BR**（CSV 或 XHR），不信订单页汇总 |
| **V4-2** | Playwright 职责 = 导航 + 点导出 + 等下载；**禁止**滚表抠 DOM 作为主路径 |
| **V4-3** | 抓包优先于 DOM，CSV 优先于抓包（稳定性排序：CSV > XHR > DOM） |
| **V4-4** | 部分源失败允许 `partial`，但必须在 UI/审计中可见 |

## 7. 风险与依赖

| 风险 | 缓解 |
|------|------|
| SC 改版导致 CSV 列名变化 | 列名映射表 + 版本化 parser；单测夹具 CSV |
| 下载按钮 selector 失效 | 多 selector + XHR 降级 |
| 广告报表需单独登录 ads 域 | 紫鸟会话内跳转；失败单独报错 |
| 7 日 vs 30 日业务争议 | 需求锁定 7 日；`period_days` 入库可配置 |
| Agent 办公机磁盘临时文件 | 下载后解析即删；路径 `backend/data/amazon-downloads/` |

### 依赖

- 紫鸟 WebDriver + Agent 在线  
- YOTO美国账号 Seller Central 已登录  
- AUTH-P0 已部署（无合成 ACOS）
