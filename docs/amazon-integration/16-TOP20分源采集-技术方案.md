# Amazon 产品 TOP20 — 分源采集技术方案（V4）

> **版本**：v4.0  
> **日期**：2026-07-13  
> **需求依据**：[15-TOP20分源采集-需求分析.md](./15-TOP20分源采集-需求分析.md)  
> **约束**：遵守 V3 数据真实性宪法；后端模式禁止 Local Demo 回退

---

## 1. 设计目标

将 TOP20 产品指标从 **「单管道 DOM 爬取」** 改为 **「多源采集 + ASIN 键合并」**：

```
                    ┌─────────────────────────────────┐
                    │   compose_product_rows (保留)    │
                    │   enrich_product_rows (保留)     │
                    └───────────────┬─────────────────┘
                                    │
          ┌─────────────────────────┼─────────────────────────┐
          ▼                         ▼                         ▼
   ┌──────────────┐         ┌──────────────┐         ┌──────────────┐
   │ 源 A: BR     │         │ 源 B: 库存   │         │ 源 C: 广告   │
   │ CSV (主)     │         │ CSV (主)     │         │ CSV (主)     │
   │ XHR (备)     │         │ SP-API (备)  │         │ XHR (备)     │
   └──────────────┘         └──────────────┘         └──────────────┘
```

**稳定性优先级**：`CSV 导出 > XHR 抓包 > DOM 解析`（DOM 仅作最后兜底，默认关闭）。

## 3. 数据源规格

### 3.1 源 A — Business Report（子 ASIN · 最近 7 天）

**页面**：`https://sellercentral.amazon.com/business-reports/detail/sales-traffic-by-asin`

**Playwright 流程**：

1. 打开 BR 子 ASIN 详情页  
2. 选择日期：**Last 7 days / 最近 7 天**  
3. 点击 **Apply / 应用**  
4. 点击 **Download / 下载** → CSV  
5. `expect_download` 保存至 `backend/data/amazon-downloads/{tenant}/{job_id}/br_child_asin.csv`  
6. 解析 CSV → 标准化行  

**CSV 列映射（US 站，容错多语言）**：

| CSV 列名（候选） | 入库字段 |
|------------------|----------|
| `(Child) ASIN` / `子 ASIN` | `asin` |
| `Title` / `商品名称` | `product_name` |
| `Units Ordered` / `已订购商品数量` | `orders_30d` → 展示 `orders7d` |
| `Ordered Product Sales` / `已订购商品销售额` | `revenue_30d` |
| `Sessions` / `会话数` | `page_views` |
| `Unit Session Percentage` / `子商品转化率` | `conversion_rate` |

**XHR 备选**（`business_report_xhr.py`）：

- 在 BR 页 `page.on("response")` 监听含 `sales-traffic`、`byAsin`、`report` 的 JSON  
- 命中则缓存 body，解析为与 CSV 相同的 `BrRow` 结构  
- 仅当 CSV 下载失败或 0 行时启用  

**废弃**：`crawlers/business_report.py` 中 DOM 主路径 → 标记 `@deprecated`，由 feature flag `AMAZON_BR_DOM_FALLBACK=0` 控制。

### 3.2 源 B — FBA 库存

**页面**：`https://sellercentral.amazon.com/myinventory/inventory`

**Playwright 流程**：

1. 打开 Manage All Inventory  
2. 点击 **Export / 导出**（或 Download inventory report）  
3. 解析 CSV  

**CSV 列映射**：

| CSV 列 | 入库字段 |
|--------|----------|
| `asin` / `ASIN` | `asin` |
| `Available` / `可售` / `afn-fulfillable-quantity` | `inventory` |
| `sku` | `sku` |
| `product-name` | `product_name`（仅当 BR 无标题时） |

**备选**：SP-API `GET_FBA_INVENTORY_SUMMARIES`（P2，需 LWA 授权）。

### 3.3 源 C — 广告（Sponsored Products · ASIN 级）

**页面**：`https://advertising.amazon.com` → Reports → Sponsored Products → ASIN report（或等价入口）

**Playwright 流程**：

1. 从 SC 跳转或直开 ads 域（复用紫鸟 Cookie）  
2. 选日期：最近 7 天  
3. 导出 CSV  

**CSV 列映射**：

| CSV 列 | 入库字段 |
|--------|----------|
| `ASIN` | `asin` |
| `Spend` / `花费` | `ad_spend_30d` |
| `ACOS` | `acos`（优先用报表值） |
| `Sales` / `销售额` | 校验用，不覆盖 BR revenue |

**废弃**：`crawlers/campaign_manager.py` DOM 主路径 → 仅保留 XHR/CSV。

### 3.4 源 D — 订单（不进入 TOP20 合成）

`crawl_orders_v3` **不再**在 `reports` / `products` scope 中用于填充 `orders_30d`。  
出库 Tab 仍可在 `insights` scope 单独采集。

## 4. 合成层改造

**文件**：`backend/python/app/amazon/composer/product_composer.py`

| 改动 | 说明 |
|------|------|
| `compose_product_rows(report, inventory, catalog, orders=None)` | `reports` scope 调用时 **orders=None** |
| `enrich_product_rows` | 保持 V3 逻辑；`period_days` 默认 **7** |
| 新增 `attach_field_sources(row, sources: dict)` | 如 `{"revenue_30d":"br_csv","inventory":"inv_csv"}` |
| 删除/禁用 | `cap_suspicious_order_metrics` 对 BR CSV 行不应清零（CSV 可信时跳过） |

**合并键**：仅 **ASIN**（10 位，`^[A-Z0-9]{10}$`）；GR 内部 SKU（`X00…`）不入 TOP20。

**TOP20 截断**：按 `revenue_30d` 降序取前 50 入库，前端 `summarizeTopProducts(..., 20)` 展示前 20。

### 5.1 `download_helper.py`

```python
@dataclass
class DownloadResult:
    path: Path
    bytes: int
    source: str  # "csv" | "xhr"

def save_download(page, click_fn, dest: Path, timeout_ms=120_000) -> DownloadResult:
    with page.expect_download(timeout=timeout_ms) as dl:
        click_fn()
    download = dl.value
    download.save_as(dest)
    ...
```

### 5.2 `crawl_pipeline.py` 改造（`reports` / `products` scope）

```python
# 伪代码
br_rows, br_diag = fetch_business_report_csv(ctx)  # 主
if not br_rows:
    br_rows, br_diag = fetch_business_report_xhr(ctx)  # 备
inv_rows, inv_diag = fetch_inventory_csv(ctx)
ads_rows, ads_diag = fetch_ads_asin_csv(ctx)

products = compose_product_rows(br_rows, inv_rows, catalog_rows=[], order_rows=None)
products = merge_campaign_ads_into_products(products, ads_rows)  # 或专用 merge_ads_csv
products = enrich_product_rows(products)
result["page_diagnostics"] = [br_diag, inv_diag, ads_diag]
result["data_quality"] = build_data_quality(br_rows, inv_rows, ads_rows, products)
```

### 5.3 `page_registry.py` scope 调整

```python
SCOPE_TASK_KEYS = {
    ...
    "reports": ["br_csv", "inventory_csv", "ads_csv"],  # 替换 br_child_asin DOM
    "products": ["br_csv", "inventory_csv", "ads_csv"],  # 轻量 TOP20-only
}
```

`fast` 模式：**对 BR/库存/广告 CSV 禁用 fast 跳过**；仅缩短无关等待。

## 11. 性能预估

| 步骤 | 耗时（估） |
|------|------------|
| BR 页加载 + 7日 + 下载 | 20–40s |
| 库存 CSV | 15–30s |
| 广告 CSV | 20–45s |
| 解析 + 入库 | <5s |
| **合计** | **1–2 min / 店铺** |

较 DOM 滚表更稳定，总耗时相当或更短（无需多轮 scroll retry）。

## 12. 安全

- CSV 含商业数据，存于 `backend/data/amazon-downloads/`，**不入 git**  
- Agent 办公机磁盘定期清理  
- 日志不打印完整 CSV 内容，仅行数与路径
