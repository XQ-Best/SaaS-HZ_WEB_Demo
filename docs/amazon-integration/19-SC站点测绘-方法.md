# Seller Central 站点测绘方法（禁止盲点）

> **根因**：在整页 `querySelectorAll('a, span')` 上匹配「下载/导出/sku」会误点页脚 **卖家 App 下载** → `sellermobileapp?ref=*_foot_*`。

## 原则

1. **先测绘、后采集**：任何新页面先跑 `discover_amazon_sc.py`，产出 sitemap + network 日志，再写下载/解析。
2. **导航作用域**：点击仅限 `main` / `kat-side-nav` / `aside nav`；**排除 footer**；**禁止** `sellermobileapp`。
3. **数据优先抓包**：BR/库存/广告以 CSV 或 XHR 为准；DOM 仅作登录与点击导出。

## 命令

```powershell
cd D:\NIUBI\SaaS-HZ_WEB_Demo\backend\python
$env:PYTHONPATH=(Get-Location)
py scripts\discover_amazon_sc.py --headed --browser-id 16505337258263
```

产出：

| 文件 | 内容 |
|------|------|
| `backend/data/amazon-discovery/sitemap-<ts>.json` | 顶栏/侧栏链接、各配置 URL 最终落地页 |
| `backend/data/amazon-discovery/network-<ts>.jsonl` | business-reports 相关 XHR |

## 代码入口

| 模块 | 职责 |
|------|------|
| `app/amazon/navigation_guard.py` | URL 黑名单、`click_scoped`、侧栏进 BR |
| `scripts/discover_amazon_sc.py` | 逐层探测 + 抓包 |
| `app/amazon/page_registry.py` | 页面任务注册（URL 来自测绘结果） |
